PROCESS MAKES PERFECT

Design overlaps art and science.
While it has aesthetic elements the process is grounded in science.

GOAL SETTING

Before going through the design process, we should first know the intend goal of a redesign. This step is really to identify the problem a design can solve. Whether is be a revnue generating or cost cutting result should also be determined in this pahse.

DISCOVERY

With the problem identified, the research and descovery phase begins. Through qualitative and quantitave data collection we build a concrete picture of the problem. This is done through interviews, surveys, web analytics tracking and so on.

DEFINED EMPATHY

With data in hand we can now properly empathize with our users without the bias of our own assumptions. We take the data and build a single user persona who we can design for. Her problems and goals are what will guide the process prototyping and ultimately implementation

PROTOTYPING

We take the best opportunity for the users experience improvement and begin building a prototyped solution. At first this is simple sketchs but through user testing becomes more and more refined until it becomes a high-fidelty verion of what we can implement in production.

USER TESTING

With a prototye we can again remove our bais through proper testing. This can be done with coworkers at low fidelity levels, but as we continue to refine our prototype we offer it to real users that can provide realistic feedback on how our design will perform in production.

IMPLEMENTATION

Now with a well refined solution for the users problem we are ready to rollout to production. This should still be done in batches of users to run a final test at scale before offering it to the entire user base. In this phase we continue to gather data that will help restart the design process.
